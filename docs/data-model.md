# Data Model

> The data types that power the prototype. Defined in TypeScript, intended to mirror Flutter's Entity layer and ultimately Salesforce's data structure.

---

## Source of Truth

All types are defined in [`app/lib/types/index.ts`](../app/lib/types/index.ts).

All mock data is in [`app/lib/mock-data/`](../app/lib/mock-data/).

---

## Core Types

### Account

The primary domain object. Represents a Salesforce Account record.

```ts
interface Account {
  id: string;                     // Salesforce Account ID
  name: string;                   // Account name
  type: AccountType;              // "corporate" | "branch" | "standalone"
  city?: string;
  state?: string;
  distanceMiles: number;          // From rep's current location
  lastVisited: Date;              // Most recent visit date
  childCount?: number;            // Corporate accounts only
  parentId?: string;              // Branch accounts only
  contactName?: string;
  contactTitle?: string;
  phone?: string;
  address?: string;
  crmId?: string;                 // Salesforce Account ID (external)
  healthScore?: number;           // 0–100, AI-computed
  annualRevenue?: number;
}
```

**Flutter Entity:** `lib/entity/account.dart`
**Salesforce Object:** `Account`

---

### AccountDetail

Extends `Account` with AI-generated content for the detail screen.

```ts
interface AccountDetail extends Account {
  lastVisitSummary: string;       // AI-generated summary of most recent visit
  ideasForThisTime: string[];     // AI-generated conversation starters
  recentActivity: ActivityItem[]; // Ordered history of interactions
  relatedAccountCount: number;    // Total children + related
}
```

**Source:** Generated by Meeting Prep Agent before visit
**Salesforce fields:** Custom fields on Account object

---

### ActivityItem

A single logged interaction on an account's timeline.

```ts
interface ActivityItem {
  id: string;
  accountId: string;
  date: Date;
  type: "visit" | "call" | "email" | "task";
  summary: string;                // AI summary or manual note
  hasTranscript: boolean;
  repName: string;
}
```

**Salesforce Object:** `Task` or custom `Interaction__c` object

---

### Interaction

The in-progress or completed capture session.

```ts
interface Interaction {
  id: string;
  accountId: string;
  startedAt: Date;
  endedAt?: Date;
  status: "recording" | "processing" | "complete" | "draft";
  durationSeconds?: number;
  transcript?: TranscriptSegment[];
  aiSummary?: AISummary;
}
```

**Salesforce Object:** Custom `Interaction__c`

---

### TranscriptSegment

A single segment of the voice transcript.

```ts
interface TranscriptSegment {
  id: string;
  speaker: "rep" | "customer" | "unknown";
  text: string;
  startMs: number;
  endMs: number;
  confidence: number;             // 0–1 from STT model
}
```

**Note:** Speaker diarization is handled by the STT adapter. Low confidence segments (< 0.7) are displayed with muted styling and flagged for review.

---

### AISummary

The full AI-generated output from a completed interaction.

```ts
interface AISummary {
  summary: string;                // Plain-English 2–4 sentence summary
  keyPoints: string[];            // Bullet points
  commitments: Commitment[];
  risks: Risk[];
  opportunities: Opportunity[];
  followUps: FollowUp[];
  suggestedCrmUpdates: CrmUpdate[];
  confidence: number;             // 0–1 overall confidence
}
```

---

### Commitment

Something agreed to by rep or customer during the visit.

```ts
interface Commitment {
  id: string;
  text: string;
  owner: "rep" | "customer";
  dueDate?: Date;
  status: "pending" | "accepted" | "rejected";
}
```

**Salesforce Object:** `Task` (when accepted)

---

### Risk

A signal that could threaten the account relationship or deal.

```ts
interface Risk {
  id: string;
  text: string;
  severity: "low" | "medium" | "high";
  status: "pending" | "accepted" | "rejected";
}
```

**Salesforce Object:** Custom risk flag on Account or Opportunity

---

### Opportunity

An upsell, expansion, or new business signal detected in conversation.

```ts
interface Opportunity {
  id: string;
  text: string;
  estimatedValue?: number;
  status: "pending" | "accepted" | "rejected";
}
```

**Salesforce Object:** `Opportunity`

---

### CrmUpdate

A suggested write to a specific Salesforce field, requiring rep approval.

```ts
interface CrmUpdate {
  id: string;
  fieldName: string;              // e.g. "Account.Description"
  currentValue?: string;          // What's currently in Salesforce
  suggestedValue: string;         // What AI recommends writing
  confidence: number;             // 0–1
  status: "pending" | "accepted" | "rejected";
}
```

## Sort Options

```ts
type SortOption = "alphabetical" | "distance" | "lastVisited" | "company";
```

| Value | Behavior |
|---|---|
| `alphabetical` | Sort by `account.name` A→Z |
| `distance` | Sort by `account.distanceMiles` ascending |
| `lastVisited` | Sort by `account.lastVisited` descending (most recent first) |
| `company` | Group by parent (sort by parentId, then name) |
